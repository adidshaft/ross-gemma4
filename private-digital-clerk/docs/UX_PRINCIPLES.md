# UX Principles

## Tone

- Calm
- Professional
- Legible under pressure
- Honest about capability limits

## Copy rules

- Use `Designed to keep case files on this device`
- Use `Works locally on this device`
- Use `Draft for advocate review`
- Use `Source-backed output`
- Use `Public-law search`
- Avoid `AI lawyer`
- Avoid `legal advice`
- Avoid blanket privacy guarantees

## Onboarding

- Minimal questions only
- Explain why Private AI Pack download exists
- Show storage, network condition, and recommendation plainly
- Hide technical model names during onboarding

## Case workspace

- Feels like a digital file room, not a chat toy
- Surface documents, timeline, issues, evidence, ask, and drafts as clear work areas
- Make source chips obvious and tappable
- Keep drafting tools subordinate to the case file

## Error handling

- Explain what failed
- State whether case files were affected
- Offer the next safe step
- Fail closed on privacy-sensitive actions

## Accessibility

- Dynamic type support
- Comfortable body sizing
- Strong contrast in light and dark modes
- Large tap targets
- Minimal motion
